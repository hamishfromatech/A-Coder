## Project Overview

A-Coder is an AI-native code editor (IDE) forked from Void Editor, which is forked from VS Code / Code - OSS. It adds intelligent AI features (Chat, Plan, Agent, Learn modes) on top of VS Code's foundation. Node.js v22 required (.nvmrc).

## Development Commands

```bash
npm install                    # Install dependencies
npm run buildreact             # One-time React build (scope-tailwind + tsup)
npm run watchreact             # Watch React with nodemon + tsup
npm run watch                  # Watch TypeScript (client + extensions)
npm run compile                # One-time TypeScript compilation
./scripts/code.sh              # Launch dev app (Cmd+R to reload)
npm run eslint                 # Lint
npm run test-node              # Node unit tests (Mocha)
npm run test-browser           # Browser unit tests (Playwright)
npm run test-extension         # Extension tests
./scripts/test.sh --glob **/someFile.test.js   # Run specific test
npm run gulp -- vscode-darwin-arm64            # Production build (macOS ARM)
```

After React changes: `npm run buildreact` then Cmd+R. After TypeScript changes: Cmd+R.

## Architecture

### Process Model (inherited from VS Code)

- **Main process** (`electron-main/`): Node.js access, can import `node_modules`. Handles LLM API calls, MCP connections, file system, settings persistence.
- **Browser/Renderer process** (`browser/`): DOM/UI access, NO direct `node_modules`. Handles React UI, chat, tool orchestration.
- **Common** (`common/`): Shared between both processes. Types, settings definitions, model capabilities, service interfaces.

Browser code that needs Node.js access uses VS Code's IPC channel system (e.g., `sendLLMMessageChannel.ts`, `mcpChannel.ts`, `settingsChannel.ts`).

### All A-Coder Code Lives Here

`src/vs/workbench/contrib/void/` — do NOT modify files outside this directory without consulting the user first.

#### CSS Scoping with `scope-tailwind`

A-Coder's React UI renders inside VS Code webviews, which already have their own Tailwind/CSS. To avoid class name collisions, all Tailwind classes are scoped with a `void-` prefix using the `scope-tailwind` tool.

**Build pipeline** (defined in `build.js`):

1. **scope-tailwind** processes `src/` → generates `src2/` with prefixed classes and `src2/styles.css`
2. **tsup** bundles `src2/` entry points into `out/` as ESM (entry points defined in `tsup.config.js`)
3. **fix-imports-after-build.cjs** post-processes `out/` JS files to fix incorrect import paths

**How `scope-tailwind` transforms classes:**

The command `npx scope-tailwind ./src -o src2/ -s void-scope -c styles.css -p "void-"` does three things:

1. **Prefixes Tailwind utility classes in JSX** — Every Tailwind class gets `void-` prepended:
   - `flex` → `void-flex`, `bg-void-bg-1` → `void-bg-void-bg-1`, `text-sm` → `void-text-sm`
   - Compound variants like `group-hover:opacity-100` → `group-hover:void-opacity-100`
   - Custom `void-*` color tokens become double-prefixed: `bg-void-bg-1` → `void-bg-void-bg-1`

2. **Wraps all CSS in `.void-scope`** — The generated `src2/styles.css` wraps every rule inside `.void-scope`, so styles only apply to elements within a `void-scope` container:
   ```css
   .void-scope .void-flex { display: flex; }
   .void-scope .void-bg-void-bg-1 { background-color: var(--void-bg-1); }
   ```

3. **Uses `content: ['./src2/**/*.{jsx,tsx}']`** in `tailwind.config.js` — Tailwind scans `src2/` (the already-prefixed files) to know which prefixed classes to include in the CSS output.

**Important:** Write Tailwind classes in `src/` files using their **unprefixed** form (e.g., `className="flex gap-2 p-4"`). The `void-` prefix is added automatically during the build. Never manually write `void-` prefixed classes in `src/` files — they'd get double-prefixed in `src2/`.

## Coding Conventions (from .voidrules)

- All types mapping A → B named `bOfA` (e.g., `toolNameOfToolId`)
- Never cast to `any` — find the correct type
- Don't add/remove semicolons unnecessarily
- Don't modify files outside `src/vs/workbench/contrib/void/` without asking
